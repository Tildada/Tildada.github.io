General practical tips for sentiment analysis

sentiment datasets/Lexica
sentiment-aware tokenizer(helpful)(like nltk.tokenize.casual.TweetTokenizer):
• preserve emoticons, social media markup(eg.tags),capitalization where it seems meaningful
• regularize lengthening(eg. YAAAAAY —> YAAAY)
• capture #$% curse words, significant multiwords expressions(eg. out of this world)
stemming:
Porter and Lancaster not recommanded.
The WordNet stemmer not that bad but don’t worth it.
Part of speech(POS) tagging:
maybe not good: some cases show that the same word with the same part of speech has diffent sentiments(ambiguous)
Simple negation marking(helpful):
Append a _NEG sufﬁx to every word appearing between a negation and a clause-level punctuation mark. (Das and Chen 2001; Pang et al. 2002)

The Stanford Sentiment Treebank (SST)

Socher et al. (2013)
SST : five way problem: 0 —-> 4 (very negative –> very positive)

sst.py

each reader is yielding a (tree, score) pair
tree[0](left child)/tree[1](right children)
a framework has been given
give it a feature function and a model wrapper, and that’s all.(test if they work)
pay attention to macro avg f1-score
sklearn.feature_extraction.DictVectorizer: turn a dictionary into a matrix

Methods: hyperparameters and classifier comparison

Hyperparameter search: util.fit_classifier_with_crossvalidation
Classiﬁer comparison: Wilcoxon signed rank test/McNemar’s test

Feature representation

Hand-built feature: Negation
Extension to other kinds of scope-taking(said/might)
Lexicon-derived features
Modal adverbs: It is quite possibly a masterpiece.
Thwarted expectations: It’s really good blalala, but….
Non-literal language: Like 50 hours long.
Assessing individual feature functions: sklearn.feature_selection
Distributed representations as features:eg. word embedding + sum
