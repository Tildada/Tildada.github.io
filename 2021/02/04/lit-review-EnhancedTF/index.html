lit review "Named Entity Recognition and Relation Extraction using Enhanced Table Filling by Contextualized Representations"

Wentao's Blog
2021-02-04
Entity and Relation Extraction, lit review

一、解决了什么问题

实体关系联合抽取可以使用填表的方法完成。以前的工作通常会设计复杂的特征来编码上下文和实体与关系之间的依赖。还会按照一定的顺序进行填表，这样基于历史预测结果的填表方式也使得并行标注难以进行。

二、如何解决

使用pre-trained BERT表示实体并编码实体间长距离依赖，以简化特征工程。
使用span-based features增强实体表示。
使用tensor点积一次填清，而非使用beam-search逐格填表。

Word Representations

Bert使用sub-word tokens，但是实体需要在单词层面上进行标注，实验尝试了使用first sub-word token / mean-pooling sub-word tokens / mean-pooling sub-word tokens and [CLS] / max-pooling sub-word tokens，最终使用max-pooling得到单词表示

Named Entity Recognition

对于某一单词的特征，由三部分连接组成：该词的表示，前一时刻的label embedding，前一段时间内(first(i-1),…, i-1) span BERT embeddings的max-pooling，first(i)表示包含词i的实体开始位置时刻。
实体识别使用全连接层+softmax

Relation Extraction

RE model的输入是entity-span表示和当前时刻的entity-label表示的连接。entity-span表示由对应实体的token embedding max-pooling组合而成。对于某一可能的关系，计算在每一时刻下的query和key vector，得到Q和K。将Q和K的点乘，使用Softmax计算得到任意两时刻组合下横跨所有可能关系的概率分布，以此一次性预测获得所有输入词对的关系标签。

三、分析结论

  1. RE错误原因：缺乏外部知识和全局约束。
  2. span-level features在NER和RE任务中都可帮助改善表现。
  3. 由于存在长距离依赖，之前的方法基本假设早时的决策有助于后期的决策，因此一般会按照一定次序顺序进行填表。但是作者在尝试后，并未发现模型性能的明显改善，推断RE可能已经通过BERT得到了长距离的依赖信息，且参考历史的预测还需要引入较多参数，增加了模型的复杂性。
  4. 尝试较长文本，将多个句子通过[SEP]分开后输入模型。NER表现提高明显，但RE无较大改善。发现NER通过捕捉coreference提高了性能，但一些对于RE很重要的实体却未能被标注成功。
