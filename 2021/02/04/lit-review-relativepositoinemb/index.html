lit review "Improve Transformer Models with Better Relative Position Embeddings"

Wentao's Blog
2021-02-04
Position Embedding, lit review

一、解决了什么问题

作者认为在现有工作中relative position未被充分利用。
论文中梳理了与relative position embedding相关的不同处理方法，希望能找到方法增加 key, query, position embeddings之间的交互。
Inductive property: Can BERT, trained on short sequences, generalize to longer sequences from the perspective of position embeddings?

二、如何解决

Quick Review

  1. Self-Attention: original transformer architecture
  2. Absolute position embedding in BERT: learnable parameters trained to maximize the log-likelihood of the MLM task, 训练前需要确定序列的最大长度n。
  3. Shaw’s relative position embedding: 作者认为相对位置信息在一定距离之外是没有意义的，引入clipped value k，即允许的最大相对位置距离。
  4. XLNet’s relative position embedding: 对Q引入bias，对距离的度量使用原transformer paper中的sinusoid formulation

论文中主要专注于同Shaw’s relative position embedding的比较

三、分析结论

Complexity Analysis

method 3需要更多的算力，method 1, 2, 4都与 absolute position embedding baseline相似

Models evaluation on datasets

SQuAD: method 4精度高，计算效率高。
GLUE: 不同方法之间没有明显的差别。可能是因为只用了第一个token [CLS]，relative position embeddings对模型的影响较小。
MRPC: method 3准确率较低，可能是因为该方法中query, key, position embedding之间的交互在数据集上不够稳定。

Models with various k

Can a BERT model trained on short sentences be generalized to handle longer sentences? 对于这个问题，可以尝试设置不同的max sequence lengths，再在更长的序列上测试。在这种情况下，relative position embedding是一个更好的选择，因为不受最大长度限制。Bert在fine-tuning时，序列长度不能超过512。作者使用relative position embedding，在SQuAD上，将Fine-tuning的最大长度设置为576, 640和704设计实验。实验发现使得F1 score提升的来源主要是那些token数量超过512个的数据。因此推测，若数据集中长文本数量更多，其增益可以更明显。

作者还设计实验，改变 clipping distance k，查看它对模型精度的影响。结果表明，尽管序列的平均长度为130，但是k>32的部分在模型训练时只能提供边际信息，再增加k便无明显的效果提升。

Relative position embeddings for large BERT models

Method 4 on a pre-trained BERT large model: boost the F1 score of 93.15 to 93.55.
We allow larger maximum sequence lengths (576, 640 and 704) in SQuAD fine-tuning task but do not observe additional gain in this case.

Relative Position Visualization

作者发现，token更多关注它们的邻居，对距离较远的token的关注几乎为零。这也解释了为什么一个小的k对于relative position embedding来说是足够的，因为超过这个范围的注意力权重接近为零。

token对自己的关注度通常接近零。这可以用MLM任务解释，因为在该任务中，模型无法看到自己，当前token的有用信息主要由其邻居提供。
